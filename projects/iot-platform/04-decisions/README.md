# iot-platform 架构决策记录

本目录用于记录项目中的重大技术决策。

## ADR 规则

文件命名：

```text
ADR-001-xxx.md
ADR-002-xxx.md
```

每个 ADR 应包含：

- 背景。
- 决策。
- 原因。
- 替代方案。
- 影响。
- 后续动作。

## 已确定决策

### ADR-001：基于 RuoYi Fast 单体版本建设平台

原因：

- 项目早期更需要稳定、快速交付和可维护性。
- 单体架构便于部署、调试和交付。
- 不为尚未明确的规模需求提前拆分微服务。

### ADR-002：后端以 Java 17 作为默认开发环境

原因：

- Java 17 是长期支持版本。
- 生态成熟，适合 Spring Boot 和工业业务系统。

### ADR-003：统一 Docker Compose 部署

原因：

- 降低部署复杂度。
- 便于本地、测试、生产环境保持一致。
- 便于配置、数据、日志外置。

### ADR-004：高频采集数据与业务数据分离建模

原因：

- 高频数据写入量大，查询模式不同。
- 普通业务表不适合承载高频时序数据。

### ADR-005：配置下发直接形成 MQTT payload

原因：

- 减少无价值中间步骤。
- 提高配置下发链路可追踪性。
- 便于网关或设备直接消费。
