# iot-platform 历史记录

## 用途

本目录用于记录项目里程碑、已知问题、技术债和历史排查结论。

## 已知重点风险

- Flowable 动态跳转、驳回、加签容易造成历史记录和业务状态不一致。
- TDengine 容器重启后可能出现恢复、leader、数据目录权限等问题。
- RuoYi 权限标识、菜单、接口映射需要保持一致。
- 设备层级和数据权限设计不能后期临时拼接。
- Docker 部署中必须谨慎处理数据卷，避免误删生产数据。

## 记录规则

每次重大问题解决后，应补充：

```text
问题：
现象：
原因：
解决方案：
影响范围：
后续建议：
关联提交：
```
