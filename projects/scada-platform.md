# SCADA 平台项目规约

## 1. 项目定位

本项目面向工业现场或飞机装配场景的可视化监控、设备状态展示、报警、交互控制和现场态势呈现。

核心目标是把复杂现场对象以清晰、稳定、可操作的界面呈现出来，而不是只做静态大屏。

## 2. 默认技术栈

前端：

- Vue 3。
- Element Plus。
- Three.js / Canvas / SVG / WebGL 按场景选择。
- WebSocket 或 SSE 承载实时状态。

后端：

- Java 17。
- Spring Boot。
- 标准 REST API。
- WebSocket 网关或消息推送服务。

部署：

- Docker Compose。
- Nginx 提供静态资源和反向代理。

## 3. 核心对象

典型对象包括：

- 飞机部段本体。
- 装配型架平台。
- 制孔机器人。
- 调姿定位器。
- 移载 AGV。
- 工位、产线、区域。

对象建模必须优先服务于状态展示、定位、报警和操作闭环。

## 4. 架构原则

- 可视化对象与实时数据解耦。
- 静态模型、布局配置、实时状态分层管理。
- 告警逻辑不得只写在前端。
- 前端动画不得依赖高频全量刷新。
- 页面性能优先考虑增量更新、节流和状态差异更新。

## 5. 数据原则

实时状态至少包含：

- 对象标识。
- 状态值。
- 状态时间。
- 质量码或有效性。
- 报警级别。
- 数据来源。

不得假设所有设备数据稳定到达。

## 6. 交互原则

SCADA 界面交互必须明确区分：

- 只读监视。
- 本地操作。
- 远程控制。
- 高风险控制。

高风险操作必须有二次确认、权限校验和操作记录。

## 7. AI 开发约束

Codex 修改本项目时：

- 不得把大量业务判断堆在一个 Vue 组件中。
- 不得用全量定时刷新替代实时数据订阅。
- 不得把报警规则只写死在前端。
- 可视化组件必须注意资源释放，避免长期运行内存泄漏。
- 涉及控制指令时必须检查权限和确认流程。

## 8. 验收要求

至少验证：

- 页面可构建。
- 关键对象能显示。
- 状态更新链路清楚。
- 异常状态和无数据状态有表现。
- 浏览器控制台无明显错误。
- Docker/Nginx 配置可运行。

## 9. 已知重点风险

- 过度追求视觉效果导致可维护性差。
- 实时刷新策略不当导致浏览器性能下降。
- 三维模型过大影响加载速度。
- 控制操作如果缺少权限和审计，存在安全风险。
