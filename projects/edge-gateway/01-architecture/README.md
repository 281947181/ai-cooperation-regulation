# edge-gateway 架构规约

## 总体架构

中心侧负责配置管理、下发、监控、数据消费和治理。

边缘侧负责协议连接、采集执行、本地缓存、断点续传和状态上报。

## 技术栈方向

后端：

- Java 17。
- Spring Boot 稳定版本。
- PostgreSQL 存储中心侧业务配置。
- Kafka 或 MQTT 承载消息链路，按场景区分。

边缘侧：

- Java 或轻量服务。
- OPC UA、Modbus TCP、S7 等协议适配。
- 配置驱动采集。

部署：

- Docker Compose。
- 配置、数据、日志外置。
- 能外挂的应用内容优先外挂。

## 架构原则

- 控制类命令优先同步直连，消息队列主要用于数据上报和异步事件。
- 协议适配必须与业务模型解耦。
- 点位模型必须可版本化、可下发、可追溯。
- 中心侧不能假设所有边缘数据实时、完整、顺序到达。
