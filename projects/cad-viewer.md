# CAD / 数模前端展示项目规约

## 1. 项目定位

本项目面向工业数模在前端页面中的展示、基础操作和轻量交互。

核心目标是让用户能在浏览器中稳定查看模型、旋转、缩放、平移、选择和进行基础状态展示，而不是在浏览器中完整复刻专业 CAD 软件。

## 2. 默认技术栈

前端：

- Vue 3。
- Three.js 或 TresJS。
- glTF / GLB 作为优先展示格式。
- Draco / Meshopt 等压缩方案按模型大小决定。

转换链路：

- 优先使用商业或成熟工具处理 CATIA、STEP、IGES 等工业 CAD 格式。
- 开源方案可用于 POC，但不能默认承诺高保真生产转换。

部署：

- Web 前端通过 Nginx 或项目统一部署方式提供。
- 模型文件通过对象存储或静态文件服务提供。

## 3. 格式原则

浏览器端优先展示：

- GLB。
- glTF。

不建议浏览器直接解析：

- CATIA 原生文件。
- 大型 STEP。
- 大型 IGES。
- 复杂装配原始 CAD 文件。

工业 CAD 到 Web 展示格式的转换应作为独立服务或预处理流程，而不是强行放到前端。

## 4. 架构原则

- 前端只负责展示和基础交互。
- 模型转换、轻量化、压缩和缓存应在后端或离线流程完成。
- 原始 CAD 文件、转换产物、缩略图、元数据应分层管理。
- 不承诺浏览器侧保留完整 CAD 参数化特征。

## 5. 交互原则

基础能力包括：

- 加载模型。
- 旋转、缩放、平移。
- 重置视角。
- 模型显隐。
- 基础选择。
- 局部高亮。
- 截图或视图保存（按需求）。

高级能力如剖切、测量、装配树、BOM、PMI、爆炸图，应单独评估实现成本。

## 6. AI 开发约束

Codex 修改本项目时：

- 不得承诺前端直接解析 CATIA 原生文件。
- 不得把 CAD 转换逻辑混入普通页面组件。
- 不得把大模型直接打包进前端构建产物。
- 不得忽略模型加载失败、超时、格式不兼容等异常状态。
- 不得为了效果引入过重依赖而不说明体积和维护成本。

## 7. 验收要求

至少验证：

- 示例 GLB 模型可加载。
- 旋转、缩放、平移正常。
- 加载中、加载失败、空模型状态有处理。
- 前端构建通过。
- 模型资源路径可配置。

## 8. 已知重点风险

- CATIA 到 GLB 自动转换存在商业授权、格式兼容和几何保真问题。
- STEP 开源转换对复杂装配、曲面、颜色、层级、PMI 支持可能不足。
- 大模型会影响浏览器内存、加载时间和渲染帧率。
- TresJS 提升 Vue 集成体验，但底层仍受 Three.js 和 WebGL 能力限制。
