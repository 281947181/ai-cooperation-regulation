## 已知风险与技术债

### Flowable 历史数据治理

关注：

- `ACT_HI_*` 表增长。
- 历史流程保留策略。
- 历史归档机制。

### MinIO 对象治理

关注：

- Bucket 规范。
- 对象命名规范。
- 生命周期管理。
- 存储分类。

目标是避免对象存储逐渐失控。

### Docker 升级与回滚能力

目标：

- 可重建。
- 可升级。
- 可回滚。
- 数据卷独立。

避免升级依赖人工经验。

### 数据库版本管理

建议建立数据库变更机制：

- Flyway；或
- Liquibase。

避免环境间结构漂移。

### 权限模型演进

需关注：

- 菜单权限。
- 角色权限。
- 数据权限。
- 项目权限。
- 流程权限。
- 附件权限。

避免后期权限体系失控。

### 前端依赖安全升级

当前 `npm audit` 仍存在中危和高危漏洞。

不得直接执行：

```bash
npm audit fix --force
```

原因：可能触发 ECharts 6 大版本升级和插件破坏性降级。应单独建任务评估升级影响。

### 后端 CORS 收敛

当前后端 CORS 策略较宽松。

开发环境可以接受，但生产环境建议单独收紧允许来源，避免长期维持全开放策略。

### 文件预览后续演进

后续可评估：

- 压缩包清单预览。
- CAD/数模图纸预览。
- 视频/音频预览及 HTTP Range 支持。
- PDF 是否长期保留 ONLYOFFICE 统一预览，或恢复原生 PDF Provider 作为可选方案。
- ONLYOFFICE 生产环境密钥轮换、备份、升级和监控。
