# 单轮任务模板

本模板用于生成具体项目中的单轮任务提示词。

具体任务 Prompt 不保存在本规约仓库中，应保存在目标项目自己的文档或任务记录中。

---

## 1. 本轮任务标题

简要说明本轮任务。

示例：

```text
完善设备管理模块的地区/厂区/厂房/产线/设备层级维护能力
```

## 2. 必须引用的规约

执行前必须阅读并遵守：

```text
constitution.md
runtime.md
projects/对应项目规约.md
```

## 3. 任务背景

说明为什么要做本轮任务，当前系统处于什么状态，前置任务是否已经完成。

## 4. 任务目标

用清晰的条目说明本轮要完成什么。

建议格式：

```text
1. 新增 xxx 能力。
2. 修改 xxx 页面。
3. 完善 xxx 接口。
4. 更新 xxx 文档。
```

## 5. 非目标范围

明确本轮不做什么，防止 Codex 擅自扩大范围。

示例：

```text
本轮不修改权限系统。
本轮不调整数据库部署方式。
本轮不重构无关模块。
本轮不升级依赖。
```

## 6. 技术约束

说明本轮必须遵守的技术条件。

示例：

```text
后端 Java 17。
前端 Vue 3。
部署使用 Docker Compose。
沿用项目现有 RuoYi 代码风格。
```

## 7. 执行步骤要求

Codex 必须按以下步骤执行：

```text
1. 同步远程仓库最新代码。
2. 检查当前分支和工作区状态。
3. 只搜索本轮相关目录和文件。
4. 给出简短执行计划。
5. 按计划修改代码。
6. 执行必要验证。
7. 检查 git diff，确认无关修改。
8. 提交所有本轮修改。
9. 按汇报模板输出结果。
```

## 8. 文件与目录限制

列出允许修改和禁止修改的范围。

示例：

```text
允许修改：
- backend/xxx
- frontend/src/views/xxx
- docs/xxx.md

禁止修改：
- docker-compose.yml
- 权限核心模块
- 无关菜单和路由
```

## 9. 验收要求

列出本轮必须满足的验收条件。

示例：

```text
1. 后端编译通过。
2. 前端构建通过。
3. 新增接口返回结构符合项目规范。
4. 页面可完成新增、编辑、删除、查询。
5. 数据权限不被破坏。
6. 文档已更新。
```

## 10. 提交要求

```text
必须提交所有本轮修改。
提交消息必须使用中文。
提交消息必须说明修改目的、主要内容和验证结果。
```

## 11. 汇报要求

Codex 完成后必须按以下结构汇报：

```text
一、任务结论

二、主要修改
- 文件一：说明
- 文件二：说明

三、实现说明

四、验证结果
- 命令：xxx
- 结果：xxx

五、影响范围

六、风险与遗留问题

七、提交信息
- Commit Hash：xxx
- Commit Message：xxx
```

## 12. 异常处理要求

如果遇到问题：

```text
1. 不要盲目扩大修改范围。
2. 不要连续猜测超过三次。
3. 明确说明失败现象、已尝试方案和下一步建议。
4. 涉及外部依赖、框架 bug、版本兼容问题时，必须查阅官方文档或可靠资料。
```
