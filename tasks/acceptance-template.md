# 任务验收模板

本模板用于 ChatGPT 或人工在 Codex 完成开发后进行验收。

---

## 1. 基本信息

```text
项目名称：
任务名称：
执行分支：
提交 Hash：
验收时间：
验收人：
```

## 2. 任务目标核对

逐条核对本轮任务目标。

```text
目标 1：通过 / 未通过 / 部分通过
说明：

目标 2：通过 / 未通过 / 部分通过
说明：
```

## 3. 修改范围核对

检查 Codex 是否只修改了允许范围内的文件。

```text
允许范围内修改：是 / 否
发现无关修改：是 / 否
无关修改说明：
```

## 4. 代码质量核对

检查内容：

- 是否存在重复代码。
- 是否存在过度设计。
- 是否存在魔法值。
- 是否破坏项目现有风格。
- 是否引入无关依赖。
- 是否可以用更少代码完成。
- 是否存在明显安全风险。

## 5. 构建与测试核对

```text
后端编译：通过 / 未通过 / 未执行
前端构建：通过 / 未通过 / 未执行
单元测试：通过 / 未通过 / 未执行
接口测试：通过 / 未通过 / 未执行
Docker 配置校验：通过 / 未通过 / 未执行
```

未执行的项目必须说明原因。

## 6. 功能验收

说明功能是否满足预期。

```text
功能完整性：通过 / 未通过 / 部分通过
异常状态处理：通过 / 未通过 / 部分通过
权限与数据范围：通过 / 未通过 / 部分通过
前后端字段一致性：通过 / 未通过 / 部分通过
```

## 7. 风险评估

记录当前仍存在的风险。

```text
风险 1：
影响：
建议：

风险 2：
影响：
建议：
```

## 8. 验收结论

```text
验收结论：通过 / 退回修改 / 暂缓合并
原因：
后续动作：
```

## 9. 退回修改要求

如果退回，必须明确：

```text
1. 必须修改的问题。
2. 不允许修改的范围。
3. 重新提交后的验证要求。
4. 是否允许继续沿用当前方案。
```
