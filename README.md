# AI 协作规约仓库

本仓库用于维护人与 AI Agent（主要是 ChatGPT + Codex）协作开发时的长期规约。

它不是具体项目代码仓库，也不是单轮任务提示词仓库。

## 仓库定位

本仓库只维护四类内容：

1. `constitution.md`：个人顶层开发规约，适用于所有项目。
2. `runtime.md`：AI Agent 运行规约，约束 Codex 等执行型 Agent 的工作方式。
3. `projects/`：项目级规约。每个项目一份或多份，描述该项目的技术边界、架构原则、目录约定、质量要求。
4. `tasks/`：任务模板。只提供生成单轮任务提示词的结构模板，不保存具体任务提示词。

## 不在本仓库维护的内容

以下内容不放在本仓库：

- 每个项目的具体业务代码。
- 每轮任务的具体 Prompt。
- 临时排查记录。
- 项目运行日志。
- 项目私有配置、密钥、证书。

具体项目内部可以单独维护 `.ai/prompts/`、`docs/ai/` 或类似目录，用于保存该项目自己的任务提示词和阶段性开发记录。

## 推荐使用方式

项目启动时：

1. 引用 `constitution.md`。
2. 引用 `runtime.md`。
3. 在 `projects/` 中建立该项目的项目级规约。
4. 在具体项目仓库中建立自己的 Prompt、任务记录和设计文档。

单轮任务开始时：

1. 先同步目标项目远程仓库最新代码。
2. 读取本仓库的 `constitution.md`、`runtime.md` 和对应项目规约。
3. 参考 `tasks/` 模板生成本轮任务提示词。
4. 由 Codex 执行具体开发。
5. Codex 完成后按任务模板要求汇报。
6. ChatGPT 或人工进行验收。

## 核心原则

先做对的事，再争取把事情做对。
